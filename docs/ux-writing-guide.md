# 한국어 UX 라이팅 가이드

> @sonmily/design-system | 작성: CPO | 2026-03-23
> 대상: BOKHAUS 시니어 앱 (65세+), 보호자 앱

---

## 핵심 원칙

1. **사람처럼 말하기** — 기계 같은 표현 금지
2. **존댓말 필수** — "~하세요", "~합니다"
3. **해결책 제시** — 오류만 알리지 말고 방법도 알려주기
4. **짧고 명확하게** — 한 문장에 한 가지 정보
5. **긍정적 표현 우선** — 금지보다 대안 제시

## 버튼 레이블

### 기본 원칙
- 동사로 시작 ("저장", "취소", "삭제")
- 2~4글자 최적
- 행동 결과가 명확해야 함

| ❌ 피하기 | ✅ 사용하기 | 이유 |
|---------|---------|------|
| OK | 확인 | 한국어 표준 |
| Cancel | 취소 | 한국어 표준 |
| Submit | 제출 / 저장 / 완료 | 맥락에 맞게 |
| Yes / No | 삭제 / 취소 | 구체적 행동 명시 |
| Delete | 삭제 | 명확 |
| Edit | 수정 | 명확 |
| Back | 이전 / 돌아가기 | 맥락에 따라 |
| Next | 다음 | 명확 |
| Done | 완료 | 명확 |
| Retry | 다시 시도 | 명확 |
| Loading | 불러오는 중... | 진행 상태 표현 |

### Modal 버튼 순서 (위험 액션)
```
[취소]  [삭제]
 ←안전   위험→
```
> 취소(안전)를 항상 왼쪽, 위험 액션을 오른쪽에 배치

## 에러 메시지

### 3요소 원칙
1. **무슨 문제인지** (명확히)
2. **왜 발생했는지** (선택)
3. **어떻게 해결하는지** (필수)

| ❌ 나쁜 예 | ✅ 좋은 예 |
|---------|---------|
| "오류" | "저장에 실패했습니다. 잠시 후 다시 시도해 주세요." |
| "잘못된 입력" | "올바른 전화번호를 입력해 주세요. (예: 010-1234-5678)" |
| "비밀번호 오류" | "비밀번호가 맞지 않습니다. 다시 확인해 주세요." |
| "네트워크 에러" | "인터넷 연결을 확인해 주세요." |
| "입력 필요" | "이름을 입력해 주세요." |
| "형식 오류" | "생년월일은 숫자 8자리로 입력해 주세요. (예: 19501201)" |

### 입력 오류 패턴
```
[필드명] + [무엇이 틀렸는지] + [올바른 예시]

"혈압 값을 숫자로 입력해 주세요. (예: 120)"
"전화번호는 숫자만 입력해 주세요. (예: 01012345678)"
```

## 빈 상태 (Empty State)

| 화면 | ❌ 피하기 | ✅ 사용하기 |
|------|---------|---------|
| 건강 기록 없음 | "데이터 없음" | "아직 기록이 없습니다. 첫 번째 기록을 추가해 보세요." |
| 알림 없음 | "No notifications" | "새로운 알림이 없습니다" |
| 검색 결과 없음 | "결과 없음" | "'홍길동'에 대한 결과를 찾지 못했습니다." |
| 목록 비어있음 | "Empty" | "등록된 항목이 없습니다" |

## 시니어 앱 특별 원칙

### 전문 용어 → 쉬운 표현
| 기술 용어 | 시니어 친화 표현 |
|---------|---------------|
| 로그아웃 | 나가기 |
| 로그인 | 들어가기 |
| 계정 | 내 정보 |
| 설정 | 환경 설정 |
| 알림 권한 | 알림 받기 허용 |
| GPS | 위치 |
| 업데이트 | 새 버전으로 바꾸기 |
| 캐시 삭제 | 임시 파일 지우기 |

### 행동 유도 문구 (CTA)
```
✅ "건강 기록 추가하기"  (구체적)
✅ "오늘의 혈압 측정하기"  (행동 명확)
❌ "시작하기"  (모호)
❌ "계속"  (무엇을 계속하는지 불명확)
```

### 확인 Modal 문구
```tsx
// 삭제 시 → 결과를 명확히
title: "기록 삭제"
content: "이 건강 기록을 삭제하시겠습니까? 삭제된 기록은 복구할 수 없습니다."

// 로그아웃 시 → 영향 설명
title: "나가기"  
content: "나가시면 다음에 다시 들어올 때 비밀번호가 필요합니다."
```

## 플레이스홀더 vs 레이블

| | 레이블 (label) | 플레이스홀더 (placeholder) |
|-|--------------|--------------------------|
| 역할 | 필드 이름 항상 표시 | 입력 예시 (선택) |
| 시니어 UX | **필수** | 선택적 보조 |
| 입력 후 | 사라지지 않음 | 사라짐 |

```tsx
// ✅ 올바른 사용
<Input
  label="전화번호"           // 항상 보임
  placeholder="010-0000-0000"  // 보조 힌트
/>

// ❌ 금지 — label 없이 placeholder만
<Input placeholder="전화번호를 입력하세요" />
```
